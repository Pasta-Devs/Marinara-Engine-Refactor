Module guide

Where code lives, what it touches, and what depends on it.

Use this page before adding files or moving logic. The right home is the one that makes the dependency direction obvious.

Top-Level Source Folders

Path Owner Touches Impact area when changed
src/app React bootstrap and shell. App providers, startup effects, layout shell. Whole UI startup, provider wiring, modal/shell behavior.
src/features Feature-owned React UI. Feature hooks, components, UI stores, feature API facades. The feature's screens and user workflows.
src/shared Frontend shared layer. Generic UI, browser helpers, shared stores, Tauri adapters. Every feature that imports the shared primitive.
src/engine React-free product engine. Domain rules, modes, generation, agents, repositories, contracts. Product behavior and every UI path using that engine service.
src-tauri Tauri host and Rust capabilities. Commands, state, storage, assets, provider transport, integrations. Native execution, data persistence, provider calls, bundling.

Shared API Modules

Files in src/shared/api are the bridge from frontend code to Tauri commands. They should be typed and narrow. If business logic starts to grow here, move it into the TypeScript engine or the owning feature.

Before moving code, check whether downstream imports should point to a public owner file, a lower-layer helper, or a capability port.
